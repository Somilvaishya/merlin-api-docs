Executive Summary — API Availability Assessment
This report documents the API connector research for the Merlin ERPNext v16 implementation covering three critical integration requirements: Auto Order Sync (orders flowing from eCommerce platforms into ERPNext as Sales Orders automatically), Live Inventory Sync (real-time stock balance push from ERPNext to all platforms), and Auto Logistics Partner Allocation (pincode-based courier assignment on Delivery Note creation). Based on research of official documentation from Frappe, Amazon, Flipkart, Shopify, Fynd/Konnect, Clickpost, and platform-specific sources, we have identified which integrations are natively available, which require middleware, and which have no public API and need custom/manual solutions.

Amazon SP-API
Selling Partner API — Native ERPNext Integration via Frappe App · ecommerce_integrations
✓ Available — Native
Integration Type
Native Frappe App
Official ERPNext Amazon connector moved out of core into a dedicated Frappe App. Available on Frappe Cloud Marketplace and via bench install.
Installation
Frappe Cloud / Self-Host
Frappe Cloud: install from Marketplace dashboard. Self-hosted: bench get-app ecommerce_integrations. GitHub: frappe/ecommerce_integrations
API Version Status
MWS Deprecated
Amazon shut down legacy MWS endpoints on March 31, 2024. All integrations now must run on SP-API. Ensure the Frappe app version supports SP-API (not old MWS).
Capability Available? How It Works Limitation
Auto Order Sync (SO creation) ✓ Amazon orders pulled via SP-API → ERPNext SO auto-created. Scheduled sync + manual trigger available. Products must be synced before Orders. Sequential sync required.
Product / Catalog Sync ✓ Pull product listings from Amazon into ERPNext Item Master. Sync products first, then orders. One-directional pull only by default. Catalog push requires custom logic.
Inventory Push to Amazon ✓ SP-API Inventory Feed — push available stock qty to Amazon. Scheduler-driven or webhook-triggered. Rate limits apply on SP-API Inventory endpoint. Safety stock buffer must be managed manually.
Customer Data Import ✓ Customer name, address, tax data pulled from Amazon orders into ERPNext Customer master. Amazon masks customer email in many regions. PII limitations apply.
Order Status Update ✓ Shipment confirmation and tracking numbers can be pushed back to Amazon via SP-API. Requires AWB to be available before confirmation push.
Returns / Refund Sync ~ Return orders visible in SP-API. Partially supported in Frappe app. May need custom handling for Credit Note automation. Return auto-matching to Sales Return DocType may need custom script.
Commission / Fee Reconciliation ✓ Account Group in ERPNext holds commissions, taxes, Amazon fees. Configured during setup. Settlement reconciliation is report-based, not real-time auto-reconciliation.
Credentials Required from Merlin / Amazon Seller Central
IAM ARN Refresh Token Client ID Client Secret AWS Access Key AWS Secret Key Country / Marketplace
Action: Merlin to register as Amazon SP-API Developer via Amazon Seller Central → generate credentials → share with ERP team.
Known Limitations
  • MWS-era integrations are now broken — must verify the installed Frappe app uses SP-API not MWS.
  • SP-API has per-endpoint rate limits (e.g. Orders API: 0.0167 req/sec). Large volume imports need throttling logic.
  • Amazon FBA orders have different fulfillment flow — stock is at Amazon's warehouse, not Merlin's. FBA vs FBM must be clearly separated in ERPNext.
  • Amazon India (IN marketplace) requires separate SP-API app registration.

Shopify Integration
Shopify Admin API + Webhook — Native ERPNext Integration via Frappe App
✓ Available — Native
Integration Type
Native Frappe App
Shopify integration is part of the ecommerce_integrations Frappe App. Same package as Amazon. Documented at docs.frappe.io/erpnext/shopify_integration.
Sync Method
Webhooks + Scheduler
Shopify sends webhooks to ERPNext on new orders, cancellations, and refunds. Scheduler handles periodic reconciliation. Near real-time order flow.
Inventory Push
Shopify Inventory Levels API
ERPNext pushes stock qty to Shopify via Inventory Levels API on each stock update. Location-aware inventory supported for multi-location Shopify stores.
Capability Available? How It Works Limitation
Auto Order Sync ✓ Shopify Order webhook fires → ERPNext creates SO automatically. Channel tagged as Shopify. Webhook reliability depends on server uptime; missed webhooks need scheduled fallback.
Product Sync (bi-directional) ✓ Items synced between ERPNext Item Master and Shopify Product catalog. Variants supported. Custom metafields on Shopify require additional mapping setup.
Live Inventory Push ✓ On stock ledger update in ERPNext → Shopify Inventory Levels API updated. Configurable by location. Shopify API rate limit: 2 requests/sec (standard) or 4/sec (Shopify Plus).
Refund / Cancellation Sync ✓ Shopify refund webhooks trigger Credit Note in ERPNext. Cancellation removes SO or marks cancelled. Partial refunds need testing — mapping to ERPNext Credit Note line items can be tricky.
Customer Sync ✓ Shopify customer profile pulled into ERPNext Customer master. Email-based deduplication. Guest checkouts create anonymous customers — may bloat Customer list.
Credentials Required
Shopify Store URL API Key API Password / Access Token Webhook Secret
Action: Merlin to create a Private App in Shopify Admin → share credentials with ERP team. Ensure webhook endpoint is whitelisted.

Flipkart Seller API v3.0
Flipkart Marketplace Seller API — Documented REST · No Native ERPNext Connector · Middleware Required
~ Via Middleware
API Status
Public API — Exists
Flipkart Marketplace Seller API v3.0 is fully documented and publicly available at seller.flipkart.com/api-docs. OAuth 2.0 authentication. REST endpoints available for orders, inventory, listings, returns.
ERPNext Connector
No Native Frappe App
There is no official ERPNext / Frappe Flipkart connector. Integration must be done via Unicommerce, Vinculum, or a custom-built Frappe App calling Flipkart REST endpoints directly.
Recommended Path
Unicommerce Connector
Unicommerce (Uniware) is the most proven middleware for Flipkart ↔ ERPNext. Has native Flipkart API v3 support including inventory sync, order pull, and return management.
Capability Available? How It Works Limitation
Order Pull (Auto SO) ✓ Flipkart API v3 has Order Search endpoint. Unicommerce polls it and pushes confirmed orders to ERPNext via their connector. No real-time webhook from Flipkart to ERPNext directly. Unicommerce polling adds ~5–15 min delay.
Inventory Push ✓ Flipkart API v3 has Inventory Update endpoint. Unicommerce / custom Frappe script pushes qty updates on stock ledger change. Known Flipkart issue: even when qty=0 is sent successfully, Flipkart may still allow orders ("zero-stock orders"). Must be handled with Marketplace Cancelled logic.
Shipping Label / Invoice Print ✓ Flipkart API returns label + invoice fetch endpoints. Label format (A4, A6, A7) configurable. Label reprint needed if customer cancels mid-pack — system must handle partial order cancellation.
Return Sync ✓ Return and cancellation status synced via API v3. Mapped to ERPNext Sales Return. Unverified orders (on-hold) at Flipkart end require special handling — no SLA clock runs during on-hold.
Direct ERPNext Integration (no middleware) ✕ Not natively available. Would require building a custom Frappe App consuming Flipkart REST API. Custom build = higher development cost and ongoing maintenance burden.
⚙ Recommended Approach for Merlin
Use Unicommerce (Uniware) as the middleware layer. Unicommerce has pre-built Flipkart API v3 integration and an ERPNext connector. This avoids building a custom Frappe → Flipkart bridge from scratch. Alternatively, Vinculum (Vin E-Retail) is a proven alternative with similar Flipkart integration capability.
Credentials Required
Flipkart Seller ID API Key API Password Location ID Flipkart User ID App ID (Developer Portal) App Secret
Action: Merlin to access Flipkart Seller API Developer Admin Portal → register app → share credentials with integration team.

Meesho Seller API
No Public Seller API · Private/Beta API via Approved Partners Only · Middleware Required
~ Via Middleware Only
Public API Status
No Public API
Meesho does NOT offer a public/open API to sellers directly. There is no self-service API key or documentation available on the Meesho Seller Panel. Meesho's seller panel (supplier.meeshosupply.com) provides order, payment, and return management via UI only.
Partner API (Private)
Approved Partners Get Access
Meesho has a private/beta Seller API (V2) provided only to approved ERP/OMS partners — EasyEcom, Unicommerce, Vinculum. Sellers cannot get direct API keys. You must request via Meesho team or use an approved third-party integrator.
  • Unicommerce: Meesho channel available, credentials via Meesho form
  • EasyEcom: Meesho API integration available
  • Vinculum: Meesho integration supported
Capability Available? How It Works Limitation
Auto Order Sync ~ Via Unicommerce middleware only. Meesho pushes orders to Unicommerce; Unicommerce relays to ERPNext. Manual order sync does not work. Only push-based orders from Meesho work. Failed orders cannot be re-fetched manually.
Inventory Push ~ Unicommerce "Facility Wise Inventory" config sends warehouse-level stock to Meesho. Meesho known issue: zero-stock orders still placed. Marketplace Cancelled penalty not applied but creates noise.
Label / Invoice Fetch ~ Label format configurable: A6, A7, Label+Invoice. Fetched via Unicommerce → Meesho API. Label format must be set correctly at channel config time; cannot change per order easily.
Return Sync ~ Return and cancellation statuses synced via Unicommerce. Mapped to ERPNext Sales Return. Unverified/on-hold orders need special handling in Unicommerce — do not accept manually.
Direct ERPNext Integration ✕ Not possible. No public API. No Frappe app for Meesho exists. Even if Meesho provides private API access, building a custom Frappe bridge is non-trivial and requires Meesho approval.
Critical Limitations — Meesho
  • No public seller API — direct ERP integration is not possible today.
  • Seller must fill Meesho's partner onboarding form to get API credentials — even through Unicommerce.
  • Meesho API v2 required — older v1 connectors in some tools are deprecated.
  • No real-time tracking data from Meesho — shipping is handled by Meesho logistics internally.
05
Zepto (Quick Commerce)
No Public Seller Order API · PO via Email + Seller Portal · Custom Email Parser Workaround
✕ No Public API
Order API Status
No Seller Order API
Zepto does NOT provide a public seller API for order ingestion or inventory sync. Zepto's publicly documented APIs are payment-related only (Payments API, PayTo API, OAuth2 for payments — not for seller order management). Q-commerce seller operations are handled via Zepto's seller portal only.
How Zepto Sends Orders
Purchase Orders via Email
Zepto sends Purchase Orders (POs) to suppliers via email — PDF or structured text. Operations team must read, decode, and create Sale Orders in ERPNext manually, OR set up an email parser to automate this flow. Zepto also uses auto-forwarding for POs.
Capability Available? Workaround / Approach Limitation
Auto Order / PO Sync ✕ No API. Workaround: Email parser in ERPNext (Email → SO automation) reading Zepto PO email format. "PO received to SO created" can be cut from 47 min (manual) to ~90 seconds with email automation. Email parsing is fragile. If Zepto changes PO format or email delivery fails, SO creation breaks. Gmail rule failures miss POs entirely.
Inventory Push (stock to Zepto) ✕ No API. Inventory update must be done manually via Zepto Seller Portal. No automated ERPNext → Zepto stock sync available. Manual stock updates create lag. Zepto dark stores may deplete faster than portal updates. Risk of PO requests for out-of-stock items.
SKU Mapping ~ Zepto uses its own Seller SKU codes. A cross-reference table must be built in ERPNext mapping Zepto SKU → Merlin internal SKU. One-time setup, 3–4 hours per platform. Product variants (e.g. 200ml vs 500ml) need separate mapping rows. Missing variant mapping causes order failures.
GST e-Invoice (mandatory) ✓ ERPNext auto-generates GST e-invoice on SO confirmation. Required by Zepto before goods acceptance at dark store. Must ensure GST, HSN codes, and tax rates are correctly configured in ERPNext Item master and Company settings.
⚙ Recommended Approach for Merlin — Zepto
Since no public API exists, the recommended path is: (1) Dedicate a mailbox for Zepto POs (e.g. qcommerce@merlin.com), (2) Build a Frappe email parser that auto-reads Zepto PO email and creates a draft SO in ERPNext, (3) Maintain a Zepto SKU ↔ Merlin internal SKU mapping table, (4) Update inventory manually on Zepto portal for now, triggered by ERPNext stock reports. This is a Q2 Phase 3 item after core go-live.

Blinkit (Quick Commerce)
No Public Seller Order API · PO via Email · Same Approach as Zepto
✕ No Public API
API Status
No Public Seller API
Blinkit (formerly Grofers, owned by Zomato) does not provide public API documentation for seller order management or inventory sync. Official public documentation is limited. Blinkit assigns its own internal item codes distinct from seller SKUs. No Frappe / ERPNext connector exists.
PO Model
Email-Based Purchase Orders
Blinkit sends Purchase Orders via email to the supplier. The operational model is: Blinkit dark store runs low → generates PO → emails supplier → supplier fulfills to dark store. POs must be fulfilled within tight SLAs. GST e-invoice mandatory before goods acceptance.
Capability Available? Workaround / Approach Limitation
Auto Order / PO Sync ✕ No API. Email parser workaround same as Zepto. Blinkit PO email → ERPNext email parser → draft SO. Must resolve SKU mapping (Blinkit code ↔ Merlin SKU). Blinkit item codes are Blinkit-assigned and differ completely from Merlin's. Mapping table is mandatory setup before go-live.
Inventory Push ✕ Manual via Blinkit Seller Portal. No API. Stock report from ERPNext → manual portal update. Manual updates create lag. Blinkit's demand forecasting generates POs regardless — mismatch can occur.
GST e-Invoice (mandatory) ✓ ERPNext India Compliance App generates IRN-based e-invoice. Required by Blinkit before goods acceptance at dark store. Must be auto-triggered on SO confirmation. Turnover above ₹10 Cr mandatory. Merlin must have IRP access and valid GSTIN configured. e-invoice setup is a separate configuration task.
Quick Commerce Priority Fulfillment ✓ ERPNext can configure routing rules to give Blinkit/Zepto SOs priority in pick list generation over D2C orders — custom Frappe logic. Requires custom Automation Rule or Frappe script. Not out-of-box.
Critical Limitations — Blinkit & Zepto (Q-Commerce)
  • No public seller API available for either platform — real-time order or inventory sync is technically not possible via API.
  • Both platforms send POs via email. Any automation must be email-parser-based, which carries reliability risk.
  • SKU code mismatch (Blinkit/Zepto internal codes vs. Merlin codes) is the #1 operational failure point reported by brands.
  • Inventory updates must be done manually in seller portals — automatic ERPNext → Q-commerce stock sync is not available today.
  • GST e-invoicing is mandatory for goods acceptance at dark stores — non-negotiable compliance setup needed before go-live.

Clickpost Logistics Intelligence
Multi-Carrier Shipping API · Native ERPNext App on Frappe Cloud · Pincode Serviceability + Courier Recommendation + Tracking
✓ Available — Native ERPNext App
ERPNext Integration
Official Frappe Cloud App
ClickPost Logistics Intelligence app is available on Frappe Cloud Marketplace. Generates shipments from Delivery Notes, handles AWB registration, and provides unified tracking.
Carrier Coverage
500+ Carriers via 1 API
Single Clickpost API integrates with 500+ courier partners globally. New carriers added within 24 hours. Covers all major Indian couriers: Blue Dart, Delhivery, Xpressbees, Ekart, etc.
Pincode Logic
Built-in Serviceability API
Clickpost Serviceability, Cost & TAT API — checks pincode serviceability, shipping cost, and estimated delivery time across all carriers. This is the engine for auto logistics allocation in Merlin's flow.
Capability Available? How It Works for Merlin Limitation
Auto Courier Recommendation ✓ Clickpost Recommendation Engine API — takes pickup pincode, drop pincode, weight, COD/prepaid, order type → returns ranked list of eligible courier partners. ERPNext auto-assigns top recommendation. Business rules (prefer courier X for channel Y) must be configured in Clickpost dashboard — not auto-learned.
Pincode Serviceability Check ✓ Serviceability API endpoint checks if drop pincode is serviced by any carrier. Returns pincode_serviceable = true/false + eligible carrier list. Must be triggered at order acceptance, not just at DN creation, to catch unserviceable pincodes early.
AWB Generation (Manifestation) ✓ Manifestation API — creates order on courier partner, fetches AWB number. AWB auto-populated in ERPNext Delivery Note field. Cancellation API available for RTOs. Must handle AWB cancellation on order cancellation in ERPNext.
Unified Shipment Tracking ✓ Track Shipment API + webhook pushes — Pickup Scheduled → In Transit → Out for Delivery → Delivered / RTO. Status normalized to Merlin's labels via "delivery mask" mapping table in Clickpost. Pull APIs on some carriers have delays up to 15–30 min. Critical milestones (Delivered) are reliable.
RTO / Return Auto-Match ✓ When Clickpost webhook fires "RTO Initiated", ERPNext can auto-create Sales Return against original Delivery Note. Custom Frappe script required to wire the webhook to Sales Return creation. Auto-match requires AWB to be stored on Delivery Note and Returns DocType — must be configured.
Shipping Label Generation ✓ Shipping Label API — generate or retrieve label from Manifestation request. Printable from ERPNext via Delivery Note print format. Label format varies by carrier. Standard A4/A6 available; thermal label requires printer-specific setup.
EPOD (Electronic Proof of Delivery) ✓ Clickpost captures POD from carriers where available. For B2B, manual POD doc attachment in Delivery Note is still recommended alongside Clickpost EPOD. Not all carriers provide digital POD. B2B must maintain manual signed receipt attachment as backup.
COD Reconciliation ✓ COD remittance data from carriers tracked via Clickpost. Invoice reconciliation for COD vs Prepaid orders manageable from unified dashboard. Remittance timelines vary per carrier. ERPNext payment entry still needs to be created manually or via custom automation.
✓ What This Unlocks for Merlin
  • Single API replaces separate integration with each courier (Delhivery, Blue Dart, Xpressbees, etc.)
  • Pincode-based auto-courier selection replaces manual courier assignment — saves 5–10 min per order at scale
  • Delivery status masking means customers see "Out for Delivery" not courier-specific status codes
  • RTO webhook auto-matching cuts return processing time significantly
  • Pincode priority rules configurable: e.g. COD orders → prefer Delhivery, Prepaid → prefer Xpressbees for Zone 2
Credentials Required
Clickpost Username Clickpost API Key Courier Partner Credentials (per carrier)
Action: Merlin to create Clickpost account → add courier partner accounts → share API key with ERP team. Define delivery mask (courier status → Merlin standard label) in Clickpost dashboard.

Shiprocket API
Alternative Logistics Aggregator · REST API · No Native ERPNext App · Can Integrate via Clickpost or Custom
~ Via Custom / Clickpost
Option
Shiprocket as Backup
Shiprocket is the leading logistics aggregator for Indian SME eCommerce. REST API available. Serves 19,000 pincodes. Can be used as a fallback or alternative to Clickpost for smaller volume channels. Clickpost actually integrates with Shiprocket — so carriers accessible via Shiprocket can still be routed through Clickpost.
Recommendation
Use Clickpost as Primary
For Merlin's scale and multi-channel requirement, Clickpost is the recommended primary logistics layer. Shiprocket can be used as a carrier on Clickpost (Clickpost has Shiprocket integration), giving access to Shiprocket's carrier network without maintaining a separate API connection. Avoids dual integration complexity.

Multi-Channel OMS Middleware — ERPNext Connector + Flipkart + Meesho + Other Platforms
⚙ Middleware Layer
What It Does
Central OMS Hub
Unicommerce (Uniware) is an Order Management System that connects multiple eCommerce platforms to ERPNext. It acts as the middleware, pulling orders from all channels and pushing to ERPNext via their connector.
Channels Covered
Flipkart, Meesho + More
Unicommerce has pre-built connectors for: Flipkart (v3 API), Meesho (API v2), Amazon, Shopify, and others. This is the recommended path for Flipkart and Meesho in Merlin's project.

Additional Subscription
Unicommerce is a paid SaaS platform with per-order or monthly pricing. This adds cost on top of ERPNext. Must be evaluated against the cost of custom-building Flipkart / Meesho Frappe apps directly.
✓ Channels via Unicommerce for Merlin

  • Meesho — API v2 (orders pushed by Meesho, inventory sync, returns)
  • Amazon — can also route through Unicommerce if preferred over native Frappe app
  • Inventory sync: facility-wise (warehouse-level) inventory updates to each channel

Fynd Konnect API
documentation.fynd.com/konnect/apis/konnect/orders — Fynd Commerce Platform Order API
~ Evaluated — Not Recommended for Merlin
What Fynd/Konnect Is
Commerce Platform, Not a Marketplace
Fynd (by Reliance Retail) is a unified commerce platform / OMS for brands. Konnect is their API layer for order management. It is primarily used by brands selling through Fynd's own ecosystem (Fynd Store, online-to-offline). It is NOT a connector to Amazon/Flipkart/Meesho/Zepto/Blinkit. The Orders API documented at the URL provided is for Fynd's own order flow — not a marketplace aggregator.
Relevance to Merlin
Low Relevance — Not Required
Unless Merlin plans to sell via Fynd's own D2C platform or physical store network, the Fynd Konnect Orders API is not applicable to this ERPNext integration project. For the channels in Merlin's flow (Amazon, Flipkart, Shopify, Meesho, Zepto, Blinkit), Fynd Konnect provides no integration. Recommend deprioritizing this connector for the current scope.
Assessment of Fynd Konnect for Merlin
The Fynd Konnect API was researched as part of this report. Based on the documentation at documentation.fynd.com/konnect/apis, it is a platform-specific order management API for brands using Fynd's commerce infrastructure — not a multi-channel aggregator for Amazon/Flipkart/Meesho. For Merlin's current channel mix, Fynd Konnect is not recommended and should not be included in Phase 1 or Phase 2 integration scope. If Merlin decides to onboard Fynd as a sales channel in future, this API would become relevant.

Pincode → Warehouse Auto-Routing
No Out-of-Box ERPNext Feature · Custom Frappe Script Required · Clickpost Serviceability API Used
⚙ Custom Build Required
What's Needed
Pincode → Zone → WH Mapping
On SO submission, the customer delivery pincode must be extracted, matched to a Zone, and the nearest warehouse with available stock auto-assigned. This requires a custom Frappe Automation Rule or Server Script with a Pincode → Zone → WH mapping table maintained as a Custom DocType in ERPNext.
How to Build
ERPNext Custom Script + Clickpost
Step 1: Custom DocType "Pincode Zone Map" (pincode, zone, primary WH, fallback WH). Step 2: Frappe Server Script on SO submit — reads delivery pincode, queries map, assigns WH. Step 3: Call Clickpost Serviceability API to confirm carrier availability at that pincode before WH assignment is finalized.
Logic Rule Buildable? Implementation Consideration
Nearest WH by Pincode ✓ Pincode Zone Map DocType → lookup on SO save → auto-assign Warehouse field. Map must be maintained by ops team. ~6,000 serviceable pincodes in India — manageable as CSV upload.
Fallback if WH = 0 stock ✓ Script checks stock in primary WH → if < order qty, assigns fallback WH from map. Multi-WH split fulfillment (partial from WH1, rest from WH2) is complex — recommend avoid for now, fulfill from single WH.
Q-Commerce → Dark Store First ✓ Channel tag on SO → if channel = Zepto/Blinkit → always assign Dark Store WH regardless of pincode map. Dark store WH replenishment must be tracked separately — standard WH reports apply.
Carrier Serviceability Check ✓ Clickpost Serviceability API called on SO confirmation with pickup pincode (WH) + drop pincode (customer). Returns eligible carriers. Adds API call latency (~200–500ms) to SO submission. Must handle API timeout gracefully.

Live Inventory Push to All Platforms
Frappe Webhook / Scheduler → Platform Inventory APIs · Custom Per-Channel Implementation
⚙ Custom Frappe Build
Platform API Available? Push Method Rate Limit / Note
⚡ Amazon ✓ SP-API Inventory Feed. Frappe Scheduler pushes qty updates. Available via ecommerce_integrations app. SP-API rate limits. FBA inventory managed by Amazon — only FBM inventory is pushable.
🟢 Shopify ✓ Shopify Inventory Levels API. On stock ledger update → webhook fires → Shopify updated. Near real-time. Standard: 2 req/sec. Shopify Plus: 4 req/sec. High-volume updates may queue.
🛍️ Flipkart ✓ Flipkart Inventory Update API v3 via Unicommerce. Facility-wise inventory push to Flipkart listing. Zero-stock bug at Flipkart — orders may still come in after 0 is sent. Handle as Marketplace Cancelled.
🏪 Meesho ~ Via Unicommerce only. "Accept by Inventory" mode — Unicommerce accepts/rejects orders based on stock. No direct ERPNext → Meesho inventory push without Unicommerce middleware.
⚡ Zepto ✕ No API. Manual portal update only. ERPNext stock report used as reference for manual update. No automation possible. Must establish manual update SOP (e.g. daily 9am stock update on Zepto portal).
💛 Blinkit ✕ No API. Manual portal update only. Same as Zepto — ERPNext generates stock report, ops team updates portal. Blinkit's demand forecasting may generate POs for items even with updated stock. Monitor closely.
Frappe Implementation
Scheduler + Webhook Pattern
ERPNext Frappe Scheduler fires on stock ledger update (on_submit hook on Stock Entry / Delivery Note) → calls platform inventory APIs in sequence. For high-frequency stock updates, a 15-minute scheduled job is more stable than real-time per-transaction calls to avoid rate limit hits.
Safety Stock Buffer
Per-Channel Threshold Config
Each platform should receive: Available Qty = ERPNext Stock − Reserved Qty − Safety Stock Buffer. Buffer prevents overselling when multiple channels sell simultaneously. Buffer values must be defined by Merlin ops team per channel and set in the integration config.
